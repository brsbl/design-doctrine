# Use a small semantic token ladder

Use a small semantic tokenized surface system rather than ad hoc spacing and color fixes so related UI stays uniform and maintainable.

## Why

One-off visual fixes accumulate into drift; a semantic ladder makes nearby surfaces feel related and changes predictable.

## Prefer

- Define a small semantic set of surface, border, spacing, and elevation roles.
- Reuse roles across related components.

## Avoid

- Fixing each component with unrelated literal values.

## Use when

- Several related surfaces need visual tuning.

## Do not use when

- A one-off visual artifact intentionally has no product-system role.

## Evidence

- Requested uniform spacing be maintained through a shared token system.

## Check

- Can each visual value be named by semantic role?
- Do related components reuse the same ladder?
- Would one token change update the family coherently?
