# Compose pattern references from reusable UI anatomy

For Pattern Atlas references, compose recognizable UI anatomy from reusable components rather than drawing bespoke abstract scenes.

## Why

Reusable component anatomy makes references more credible, coherent, and maintainable than one-off drawings that only approximate a real interface.

## Prefer

- Build reference compositions from reusable controls, containers, and layout primitives.
- Preserve recognizable interface anatomy while keeping the visual treatment intentionally neutral.

## Avoid

- Bespoke scenes that look like generic diagrams instead of usable interface patterns.
- Duplicating component anatomy separately for each reference.

## Use when

- A Pattern Atlas reference must represent a reusable product pattern.

## Do not use when

- The requested artifact is a deliberately abstract diagram rather than a UI reference.

## Evidence

- Requested Pattern Atlas wireframes be built from components and verified with screenshots because the existing references were not built credibly enough.

## Check

- Does the reference use reusable UI anatomy rather than a bespoke scene?
- Is the represented pattern recognizable at gallery and inspector scale?
- Do screenshots confirm the composition remains readable in its target surface?
