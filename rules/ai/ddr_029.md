# Use one coherent model for agent context

Context pills, images, and threads should compress and expand through one coherent model across the composer and action timeline.

## Why

Context changes meaning when each surface summarizes, expands, or orders it differently.

## Prefer

- Use the same ordering, summary, count, and expansion logic wherever agent context appears.

## Avoid

- Independent context models for composition and history.

## Use when

- The same agent context appears in more than one lifecycle surface.

## Do not use when

- A surface intentionally represents a different immutable historical snapshot.

## Evidence

- Requested context pills and images collapse compactly using a model consistent with the command palette, especially when context is absent.

## Check

- Does the same context retain identity across surfaces?
- Are compression and expansion rules consistent?
- Can the user tell what was actually sent?
