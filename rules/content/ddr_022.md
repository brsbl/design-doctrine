# Describe the interaction that actually ships

UI copy must name the real gesture and outcome, especially hover versus click, rather than describe an idealized interaction.

## Why

Incorrect instructions make a working feature feel broken and invalidate documentation evidence.

## Prefer

- Verify and name the exact shipped gesture and result.

## Avoid

- Describing click behavior when the feature uses hover, or vice versa.
- Copy based only on intended behavior.

## Use when

- Copy instructs the user how to interact with a feature.

## Do not use when

- The text is explicitly conceptual and labeled as such.

## Evidence

- Corrected feature copy to describe clicking, browser tabs, and interactive pills, while removing redundant explanation.

## Check

- Does the described gesture match the current rendered product?
- Does the stated outcome occur?
- Was the interaction verified directly?
