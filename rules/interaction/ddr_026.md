# Focus direct editing on the real anchor

Direct edit should focus the intended object immediately, make its anchor obvious, and suppress transient hover or selection interference.

## Why

Editing feels direct only when the user remains anchored to the object and input focus is unambiguous.

## Prefer

- Move focus immediately to the real editing target.
- Keep the source anchor visible and quiet competing hover state.

## Avoid

- Opening an editor without focus.
- Leaving selection or hover state that visually competes with editing.

## Use when

- A user invokes editing directly from an object.

## Do not use when

- The edit requires a deliberate multi-step modal workflow for safety.

## Evidence

- Reported editing a formula pill selected its container and flashed a hover card; asked for a clearer editing anchor.

## Check

- Does focus land on the intended input?
- Can the user still identify the source object?
- Are competing transient states suppressed?
