# Navigate to the live source object

A list item representing in-document content must navigate to the live anchored object rather than an unreachable surrogate.

## Why

Users inspect the source object to understand and act; a detached representation breaks that mental model.

## Prefer

- Anchor navigation to the real editable or inspectable source object.

## Avoid

- Navigating to a detached copy, placeholder, or unreachable location.

## Use when

- A secondary list represents content that exists in a primary document or canvas.

## Do not use when

- The source object no longer exists and the UI explicitly presents a historical record.

## Evidence

- Required comment-list rows to navigate to the actual comment anchor in the editor gutter.

## Check

- Does activation reveal and focus the real source object?
- Is the destination recoverable when the source is missing?
