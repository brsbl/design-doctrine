# Make controls behaviorally honest

A visible control must work and its pressed or selected appearance must reflect real state; otherwise remove, disable, or fix it.

## Why

False affordances and decorative state destroy trust faster than a visibly unavailable action.

## Prefer

- Wire every visible control to real behavior.
- Reflect real selected, pressed, loading, and disabled state.

## Avoid

- Inert controls that look actionable.
- Pressed styling disconnected from actual state.

## Use when

- A surface presents an interactive control or stateful action.

## Do not use when

- A clearly labeled static demonstration is intentionally non-interactive.

## Evidence

- Reported a bulk-resolution control that did nothing and looked permanently pressed.
- Rejected a nonworking upvote and a navigation dropdown that opened on unintended hover.

## Check

- Can every visible control be activated as expected?
- Does visual state match application state?
- Is unavailable behavior clearly disabled or removed?
