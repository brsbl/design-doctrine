# Treat empty state as semantic absence

Treat emptiness first as a structural decision: remove or collapse a section with no useful next action; when the surface must remain, explain what belongs there and provide the first useful action in a compact empty state.

## Why

Design every user-visible state, but do not preserve an unnecessary container merely to render a placeholder. If absence matters, the remaining state must orient the user and support progress.

## Prefer

- Collapse irrelevant empty containers so no artificial empty state remains.
- When the surface remains, explain what belongs there and keep the first useful action compact.

## Avoid

- Showing an empty card or zero-count section merely because the container exists.
- Leaving a visible empty surface without orientation or its first useful action.

## Use when

- A proposed or existing surface has no items or content, requiring a decision whether the surface should remain visible.

## Evidence

- Asked empty sections to disappear rather than display a placeholder.
- Changed external-folder emptiness to remove the folder, not show an empty container and prominent action.
- Flagged an empty state that concealed a necessary keyboard shortcut.

## Check

- Does the empty container help the user act or understand state?
- If not, can it collapse entirely?
- If guidance remains, is it compact and actionable?
