# Keep hover effects local

Hover styling and action changes belong to the owning row or control rather than an unrelated whole card.

## Why

Local hover scope makes the interactive target and ownership legible without lighting up unrelated content.

## Prefer

- Apply hover feedback to the smallest owning interactive region.

## Avoid

- Changing an entire card when only one row or action is interactive.

## Use when

- A nested action or row needs hover feedback.

## Do not use when

- The entire card is one interactive target.

## Evidence

- Requested only the hovered button, not its entire card, change state.
- Requested row actions and the drag handle appear only on row hover.
- Restored row-hover action buttons after they disappeared.

## Check

- Does the hover region match the actual click target?
- Does unrelated content remain visually stable?
