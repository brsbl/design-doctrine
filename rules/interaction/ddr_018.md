# Represent state once on the owning control

Encode installed, running, paused, and similar state once as a distinct state of the owning control; do not repeat it across rows or add competing toggles.

## Why

Duplicated state labels and parallel controls create ambiguity about ownership and the next available action.

## Prefer

- Let the owning control communicate current state and next action.
- Keep one canonical status location.

## Avoid

- Repeating the same state label throughout a row.
- Adding a second toggle that competes with the primary state control.

## Use when

- A resource has a persistent state and a related action.

## Do not use when

- A separate status is needed for monitoring at a different scope.

## Evidence

- Requested installed state be represented on its button rather than as a card-wide hover replacement.
- Rejected duplicated installed and paused state labels across related resource rows.
- Asked repeated run-status wording be removed when an icon already communicates the status.

## Check

- Where is the canonical state shown?
- Does the owning control reveal the next valid action?
- Is the same status repeated without adding information?
