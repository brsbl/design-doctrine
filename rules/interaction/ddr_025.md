# Preserve object identity during in-place editing

Editing an item in place preserves its position and commits back to that same item rather than moving the work into a separate composer or reordered draft.

## Why

Direct editing should maintain object identity and spatial context so the user knows exactly what will change.

## Prefer

- Edit and commit the same object at the same location.
- Preserve ordering unless reordering is the explicit action.

## Avoid

- Copying an item into a detached composer.
- Moving the item because it entered edit mode.

## Use when

- The user invokes edit on an existing object.

## Do not use when

- The product explicitly creates a new derived object and communicates that transition.

## Evidence

- Required in-place editing to update the queued item where it is, rather than move content into a separate prompt.

## Check

- Is the edited object still the same object?
- Does it remain in place?
- Does commit update that exact object?
