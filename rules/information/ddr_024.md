# Remove redundant metadata

Do not repeat status, labels, or explanatory metadata already clear from the surrounding object; show only what changes a decision.

## Why

Repeated metadata lowers information density and makes important differences harder to scan.

## Prefer

- Show metadata that changes interpretation or the next action.
- Choose one canonical location for each status.

## Avoid

- Repeating labels or status already conveyed by the owning control or section.

## Use when

- A dense surface repeats descriptive or status text.

## Do not use when

- Redundancy is required for accessibility, safety, or cross-context orientation.

## Evidence

- Asked run-state text be removed when icons communicate it and required overview context to carry into detail rows.
- Rejected duplicate status labeling and unnecessary inline detail navigation.

## Check

- Does each metadata item change a decision?
- Is the same status shown elsewhere?
- Can a label disappear without ambiguity?
