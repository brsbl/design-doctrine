# Use icons only when they preserve meaning

Replace text with an icon only when the symbol is semantically exact; retain essential quantity or state and avoid decorative emoji.

## Why

Compactness helps only when users can still understand the object, state, or action without guessing.

## Prefer

- Use a canonical symbol with an accessible name.
- Retain counts or state when they change the user's decision.

## Avoid

- Decorative emoji in product chrome.
- Ambiguous icons that erase essential state or quantity.

## Use when

- Text is being replaced to reduce density.

## Do not use when

- The concept is unfamiliar or has no precise established symbol.

## Evidence

- Requested semantic attachment and chat icons retain necessary counts, while row borders become lighter.
- Rejected a decorative emoji and requested feature visuals use one real, consistent example without a floating toolbar.

## Check

- Is the icon semantically exact?
- Does removing text hide quantity or state?
- Is the control still accessible without hover?
